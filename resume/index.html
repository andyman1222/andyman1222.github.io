Research

Randomness and Uniformity in Computer Software: A Brief Study

A research paper developed for the Human-Computer Interaction course at the University of Utah. Paper was developed as a class project, and as such is not publicly available.

Citation:
Andrew Herbert and Fnu Harshit. 2024. Randomness and Uniformity in Computer Software: A Brief Study. In CS 6540: Human-Computer Interac- tion, Fall 2024. Not published, 9 pages. Available upon request.

Multiplayer AI/player behavorial study

A class project developed into a component of Quantonium's The Laser Games. Objective was to develop new game AI to mimic how players of laser tag move and behave, by analyzing reference material and playtests. While the AI was successfully developed based on patterns observed from laser tag material, playtests and a cumulative paper were not finalized.

The source code is planned to be cleaned up into a Unreal Engine plugin, and planned to be released open source at https://github.com/quantonium.

Matchmaking simulation for distributed systems

A class project currently in progress, working to develop an architecture based on multiplayer game matchmaking, to assign systems to servers with workloads based on set criteria. Working on a proposed matchmaking algorithm which attempts to create matches based on an order of rulesets.

Current progress: in experimentation

Software

With Quantonium

LaserGraph

  • LaserGraph is a stats visualization webpage for The Laser Games. Players can save game stats in a ".ljson" file, which can then be processed locally on a website.
  • Features a leaderboard, hit timeline, performance hexagon, tagging network graph, by-event score progression, and hit sensor target distribution chart.
  • Clicking on some graphs opens a fullscreen view with an additional visual and more information
  • Planned features include filtering by player and team, improved time range selection, and paper printing support.
  • Website: https://laserstats.quantonium.net/

Qeng

  • Game engine written in Javascript/webGL. Latest changes include modular JS backend and complex physics animation simulation (unstable)
  • Sophisticated lighting- support currently for up to 64 lights, with customizable parameters to cast on materials differently, and negative and translucent lighting
  • Postrpocess layer- supports 10+ buffer inputs
  • Gameplay update mechanics- tick-based updating with optional customizable functions to be called before or after tick/graphics/input
  • Buffered input controls- Handle all inputs by the player in the order they arrive, not missing any inputs
  • Separate graphics thread- Updates visuals independently from game logic
  • Demos (old): https://andyherbert.net/engine/ (modular and physics simulation demos not yet available publicly)
  • Demo + engine source code: https://github.com/quantonium/Q-Engine

RoseGamesOnline

  • Rose Games Online is a website which uses a NodeJs WebSocket server to allow users to program and play games in Javascript, with HTML Canvas graphics.
  • Created for a class in a group of 2.
  • Uses Firebase Firestore backend to host and store user code. Implemented with custom JS serialization, to prevent potentially malicious and unwanted JS functions from running server-side.

TCP vs. UDP tester

  • server+client program which tests real-world performance (RTT and reliability) of TCP and UDP, generating a CSV with the results. Made for WinSock in C++.
  • https://github.com/andyman1222/speedtestnetthing

snakeInC/snakeInPy

  • Multiplayer snake-like game written in C for WinSock, and later in Python, as example starter code for network implementation.
  • Terminal-based with curses.
  • Original C-based implementation uses stateful item updates, sending the latest state of all items. A max of items and snakes is defined so game does not have to allocate heap space.
  • https://github.com/andyman1222/snakeInC https://github.com/Andyman1222School/SnakeInPy
